问题讨论

第一节：中断、异常与系统调用

Q0:中断、异常和系统调用有何异同之处？

[cx]
同：都是cpu的响应机制
异：
中断是外部设备触发、
异常是应用程序运行出错被动触发、
系统调用是应用程序主动触发、

Q1:高级语言(如 C/C++) 中的函数调用与 RISC-V 中的调用指令有何区别？

[cx] 
函数调用 : 遵循 api 约定 使用栈存储交换信息
RISC-V : 遵循 abi 约定 使用寄存器存储交换信息
Q2:如何理解调用规范中的“调用者保存”和“被调用者保存”？
[cx]:
调用者保存: 说明这些寄存器、你不记录、别人用完了、不一定还是原来那些数
被调用者保存: 说明 这些 寄存器 不管你怎么使、还回去的时候、要跟你刚拿到手时一样

调用者保存: which 寄存器 在 执行调用的函数时 前后 会被改变、需要提前保存
被调用保存: which 寄存器 在 执行调用的函数时 前后 不会被改变

Q3:同样需要进行上下文环境保存与恢复，你认为函数调用和 Trap 处理有何不同？

[cx] 
函数调用 ： 
全程在用户态
使用栈来保存与恢复
Trap:
trap从用户态切换到内核态、执行完在回到用户态
寄存器来保存与恢复

Q4:RISC-V 是否支持中断嵌套？如果不支持，应怎样做？

[cx]  支持、我猜支持、这么大的一个架构、不能嵌套都不支持吧、
中断处理时、先把寄存器们都压栈、因为新来中断会再次保存内容到寄存器就覆盖了、恢复时也先把栈内存储恢复到寄存器、再返回、

第五节：系统调用

Q1:函数调用与系统调用有何区别？

[cx]
函数调用 ： 
1.不需要切换特权级
2.只有一个栈
系统调用:
1.需要进入内核态
2.需要有栈的切换
